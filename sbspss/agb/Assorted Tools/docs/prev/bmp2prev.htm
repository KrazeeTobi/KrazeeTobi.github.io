bmp2prev [Tool Command]

Creates a script in order to preview an image.

Syntax

bmp2prev [-c] bitmap bitmap ...

Description

Creates a script in order to preview one or a number of bitmaps. The width and height of the image must be 8, 16, 32 or 64 dots, though the width and height do not need to be the same size. This command is called from imgview.sh.

The command arguments are described below:

input-file
Specifies the input bitmap file.
-o output-file
Specifies the name of the output file.
-c color
Sets the background color.

The background color can be described in one of three ways:

(1) By describing the red, green, and blue elements.
For example, red is 255,0,0 and cyan is 0,255,255.
There must not be any spaces before or after the commas.
Values of 0 can be omitted, so you could also write
red as 255,, and cyan as ,255,255.

(2) By using a 6-digit hexadecimal, starting with 0x.
For example, red is 0xff0000, green is 0x00ff00
and blue is 0x0000ff.

(3) By using a single word to describe the color. For example, the word for the color "red" is red.
The following 8 words are presently supported:
black, white, red, green, blue, yellow, purple, and cyan.
Only lower-case text is supported.

Notes

Be careful about the restrictions on bitmap size. This command is for creating a script, so you cannot directly preview the image. You must use prevcl to convert the output file into a C language source file, then incorporate this into the Previewer source files to create an AGB execution file. These tasks are executed inside imgview.sh.
