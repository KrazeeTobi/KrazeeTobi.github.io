prevprep [Tool Command]

The script preprocessor.

Syntax

prevprep [-o/-D] script-file

Description

The C language-like preprocessor. Supports the following tokens:

#include "filename"
#define
#if
#ifdef
#ifndef
#else
#endif
/*Comment*/
//Comment

However, you cannot continue to the
next line by using \ with #define etc.

The command arguments are described below:

script-file
The text file describing the script. This file is analyzed, corrected to match the AGB format and then output as a C language source file.
-o output-file
Specifies the name of the output file. If this argument is not specified, it is generated from standard output.
-D define_str
Sets the #define define_str. For example, if you write -D DEBUG then DEBUG is set and #ifndef DEBUG ~ #endif(#else) will be ignored. If you want to DEFINE two or more, then you need to write it this way: -D aaa -D bbb etc.

Notes

With comments, everything enclosed inside /* ~ */ is processed as a single group, so /* aaa /* bbb */ ccc */ is all treated as a comment (whereas in C language, ccc */ would be left over and become an error).
The same holds for #if: you can specify any amount and it will be processed appropriately.

If you want to view the same animation in separate files, you can write #include "animation filename" so you do not have to describe the same animation over and over.

Also, if you want to preview the same character with changed animation, it is convenient to write something like the following:

#ifdef ANIM1
... (Animation script)
#endif
#ifdef ANIM2
... (Animation script)
#endif
...
