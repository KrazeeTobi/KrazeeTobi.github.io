Revision History


(Version 1.04 1.05)
1.Enabled the use of some sound code in the system ROM. Added the object file which is used with "m4aLibUSC.o".
2. The panpot specification of the rhythm set is no longer affected by  sound produced by other players.
3. Made an execution version of high speed internal RAM .
4. Enabled one to limit the processing time of m4aSoundMain() during each frame.
5. Enabled the registration of song data number 1999.
6. Made it possible to set the loop position for each track with the MIDI control check number.

(Version 1.03c 1.04)
1. Fixed problem with interval when value was zero with the functions m4aMPlayModDepthSet() and m4aMPlayLFOSpeedSet().
2. Fixed problem of not being able to set the sweep value to ineffective with compatible sound 1.
3. Fixed abnormality due to fade out with volume of part where tremolo used.
4. Changed the function name for MPlayAllContinue() to m4aMPlayAllContinue().
5. Added the function m4aMPlayImmInit(), which immediately initializes the various tracks of the song you plan on starting to play.
6. Fixed problem of abnormal operation due to timing when interrupting with m4aSoundVSync() during change of direct sound playback frequency.
7. Made so you can separate the segments of the program executed with RAM from other work areas with linker script.

(Version 1.03 1.03c)

1. Fixed calculation of total size display for simple monitor.
2. Fixed problem with value for m4a_mplay???.stat when performance ends.
3. Changed compatibility of MEMACC, PRIO with MIDI Control Change Number.
4. Changed addition value of MIDI Undefined Control Change Number to avoid redundance from 20 to 30.

(Version 1.02 1.03)

1. Fixed problem of vibrato not functioning properly.
2. Added display of total size of all sound objects to simple monitor.
3. Added display of number of CH currently being used by direct sound with the simple monitor.
4. Changed the addition value used with this system's MIDI Undefined Control Change Number from 88 to 20.
5. Added the MEMACC instruction to the song data. Added memacc_s to "mks4agb.ini".   


(Version 1.01 1.02)

1. Fixed the problem of incomplete initialization of the Music Player work area.
2. Fixed rare problem of an infinite loop occurring if a new song was started on the same Music Player during chord play on the same track.
3. Made possible to create code that uses only 1 channel DMA. Added object file used with this.
4. Reduced the number of stacks used for the function m4aSoundMain().
5. Changed so that .aif files without INST-chunk can be converted now.
6. Changed the spelling of pan from "pam" to "pan".


(Version 1.00 1.01)

1. Upgraded the Simple Monitor. The data files of the Simple Monitor "SndMonDat.s" and "SndMonDat.o" were output.
2. Fixed the bug of not running if the long name of the directory was used in the directory for this system.
3. Fixed the bug of the values of Attack, Decay, and Release for CGB compatible sound not working normally for the tone color settings in "mks4agb.ini".
