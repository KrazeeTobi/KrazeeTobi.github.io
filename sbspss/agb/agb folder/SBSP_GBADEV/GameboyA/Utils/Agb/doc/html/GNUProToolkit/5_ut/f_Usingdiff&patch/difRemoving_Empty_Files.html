Removing Empty Files

Sometimes when comparing two directories, the first directory con-tains a file that the second directory does not. If you give diff the '-N' or '--new-file' option, it outputs a diff that deletes the contents of this file. By default, patch leaves an empty file after applying such a diff. The '-E' or '--remove-empty-files' option to patch deletes output files that are empty after applying the diff.
