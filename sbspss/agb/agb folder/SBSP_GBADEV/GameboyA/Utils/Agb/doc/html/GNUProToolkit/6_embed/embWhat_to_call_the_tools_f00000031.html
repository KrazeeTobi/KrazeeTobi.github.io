What to call the tools for Hitachi SH 
Cross-development tools in the GNUPro Toolkit are normally installed with names that reflect the target machine, so that you can install more than one set of tools in the same binary directory. The target name, constructed with the --target option to configure, is used as a prefix to the program name. For example, the compiler for the Hitachi SH (called simply gcc in native configurations) is called sh-hms-gcc.

Likewise, the Hitachi SH-configured debugger for GDB is called sh-hms-gdb.

For DOS-hosted toolchains, the tools are simply called by their standard names, such as gcc, gdb, etc.
